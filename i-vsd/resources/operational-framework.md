# Operational Framework

## Review Areas

- Support response expectations and respectful communication.
- Critical issue handling, including a 72-hour target where explicitly adopted.
- Security audits and vulnerability reporting.
- Incident response, user notification, correction, and restitution.
- Backup, restore, and recovery testing.
- EOL notices, migration tools, export support, and shutdown continuity.
- Lifetime or grandfathered commitment handling.
- Open-sourcing consideration if abandonment would harm users.

## Review Questions

- Which public promises require operational capacity?
- Are there runbooks, owners, severity levels, and response times?
- Are incidents reviewed for affected stakeholders and moral repair, not only uptime?
- Are users able to leave, recover, appeal, or correct harms?
- Are backups and recovery tested rather than assumed?

## Evidence Expectations

Runbooks, support SLAs, incident reports, vulnerability policy, audit records, backup restore logs, EOL plan, migration docs, user notices, and postmortems.

## Anti-Patterns

Promising reliability without operations, hiding incidents, no vulnerability path, untested backups, EOL without export, and lifetime promise reversals without remedy.
